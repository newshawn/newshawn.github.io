语法基础课

22 年 11 月 19 日开始刷 Acwing 的语法基础课，重温 c++。

第一章

  1. scanf 和 printf 在速度上比 cin cout 要快，但是 scanf 在读取字符串的时候，需要中间加一个空格来过滤空格，如 scanf("%s %s",&a,&b); 但是 cin 在输入的时候会自动过滤掉空格。前者用 cstdio，后者用 iostream，并且后者必须使用 namespace std。cin、cout 可以同时输入/输出不同类型的数据，如整型和字符串。

  2. float 和 double。

    • float 是浮点型，double 是 double float 是双精度浮点型。float 的有效位数为 6-7 位，第 7 位不一定有效，前六位一定有效，如float a = 1.2345668; 的结果为 1.234567。double 有效位为 15-16 位。99% 的情况都是用 double。
    • 关于用 scanf 来读取。键盘将输入的数据会进入缓冲区，double a;scanf("%lf,&a");然后 scanf 通过 % 来读取，比如 scanf 用 %lf 时，系统会按照 8 字节来读取，然后后面 &a 取变量地址，再将读取的数据输入到那个地址上。%f 默认读取 4 个字节，%lf 默认读取 8 个字节，如果用 %f 给 double 变量读取数据，就会将 4 字节的数据放在 8 字节的地址空间里。又因为 double 和 float 都有各自的数据排列格式，包含符号位、阶码、尾数、长度，不像 int 和 long 的关系那样，如果按照 float 的格式来读取到 double 中，那些阶码、尾数都错位了，到时候还得要强转类型。同样，如果用 %lf 读到 float 中，不仅格式错位，存储空间也不够，会有数据丢失，强转类型也没救了。
    • 用 printf 输出printf 传参的时候，c 会自动将 float 类型参数转换为 double 型，所以输出的时候问题不大。输出的时候用 %.12f也没问题，照样可以给 double 变量输出 12 位小数

  4. 如果两个类型不一样的变量相互计算，结果就会是精度更高的类型，因为精度更高的类型能兼容精度更低的类型。还有 char 类型和 int 型相加为 int ，char 和 char 相加也为 int 型。如果 int 遇到 double，就会先把 int 转换为 double。整数也是一个特殊的浮点数，把整数读进浮点数没有问题。

写题时经常犯的错

  • 浮点数的计算：最开始写的是 coneys/sum*1.0，以为这样就可以变成正确的浮点数结果了。后面就出错了，反复尝试后发现：这个是按照顺序从前往后计算的，先计算 coneys/sum，两个 int 型相除，然后 coneys 还小于 sum，结果保存在 int 中，则是 0。然后再进行下面的 *1.0 计算，0 * 1.0，会将 int 和 double 计算，会将 int 先转为 double，就类似于 0.0 * 1.0，算出来就是 0.0，最后保存在 double 中。因此，以后如果有类似的情况，就把 1.0 放在最前面。Acwing链接
  • 要注意负奇数取模的时候，判断用 if(i%2 == -1)，正奇数则是 1，写题的时候判断奇数只考虑了正奇数，没有考虑到负数。如果只判断奇数，那么用 if(i%2 != 0) 会更好。Acwing链接
  • pritnf 输出 % 的时候，要用 %%，因为百分号是转义字符，如果只用一个 %，输出是无法显示的。Acwing 链接
  • 小数没有取余计算，如果要取余就先转化为整数，可以乘 10^n。
  • 如果题目是先输入一个值 n，然后接下来要输入 n 对数值，可以使用 while(n--) 语句来对每一组数值进行处理。最开始还想，用 n-- 不就只能处理 n-1 次吗，后面想起来这是先运算再减，先运行 while(n),再 -- 。
  • 又遇到了个算完全数的问题，即一个数除了本身以外的公因数，加起来等于本身，如 6 = 1+2+3，因此 6 是一个完全数。一般这种题目都是要算很多个数，然后每个数都特别大，我一开始的思维就是暴力解法，循环算每个数，然后每个数用取模来找公因数，这样做就超时了。题目最大有 100 个数，每个数最大有 10^8，这样相当于循环就要 10^10 次了。所以可以缩小内层循环的范围，从 1 开始到数的开方结束。因为约数是一对的，一个小的对应一个大的，如 6 里面 的 2 和 3，所以只要从 1 到开方就可以找到所有小的约数。小的约数的最大可能值就是开方值，像 9、16 这种数的平方。因此这样就可以大大缩小循环次数。然后还有一个坑，就是如果你把 1 算进去，后面还会计算出对应的公因数，即数的本身，但是题目不允许将数本身加进去，所以要把 1 单独拿出来，这样就好了。以后遇到质数、完全数这种涉及到约数的，优化方面可以开方数思考。别人的题解 自己的题解
